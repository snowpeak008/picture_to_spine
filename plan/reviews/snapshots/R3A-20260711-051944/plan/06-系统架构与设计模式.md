# 系统架构与设计模式

## 1. 文档目的

本文档定义 Flash to Spine Production Assist 的逻辑边界、进程边界、模块依赖、核心接口和可进化方式。其目标不是展示技术名词，而是保证实现期间任何一个前端页面、AI 算子、导出器或存储实现都不会成为项目事实的第二来源。

本架构服务于已冻结的产品边界：闭源商业 Windows 桌面工具、本地单用户、只接收图片和动作文字、禁止第三方公有云、可选用户自托管私有远程 GPU，V1 首个集成目标仅为 Spine Editor 4.2.43。

依 `F2S-ADR-REGISTRY-001`，`F2S-ADR-ARCH-001` 是24号跨领域ADR的只读投影；`F2S-ADR-ARCH-002`至`005`是本文件拥有的implementation-local ADR canonical body。24号只登记后四项的exact owner和状态，不复制其正文。

## 2. 架构驱动因素

| 驱动因素 | 对架构的强制影响 |
| --- | --- |
| 人工门禁 | 母版、AssetSpec、分层、Rig、关键姿势、命中帧和导出必须是可持久、可审计的状态，不得只存在于 UI |
| 8GB GPU 降级 | 本地高成本模型不能是核心工作流的必选前置；手工和轻量推理必须可完成同一业务状态迁移 |
| 禁第三方云 | WebView 无任意网络能力；只有 Rust 私有远程适配器可以连接用户明确信任的端点 |
| Spine 许可边界 | 内部预览使用 Rig IR；发布包不链接、不复制、不捆绑 Spine Runtime；不直接写 `.spine` |
| 闭源商业发布 | 代码、模型权重、字体、安装器和转码器必须独立进入许可审计 |
| 可恢复长任务 | 业务工作流与后台 Job 生命周期分离，只在明确的阶段边界恢复 |
| 大图和二进制产物 | IPC 只传元数据和受控路径，不在 JSON 中传 Base64 图片 |

## 3. 架构决策

| 决策 ID | 决策 | 理由 | 不允许的替代做法 |
| --- | --- | --- | --- |
| `F2S-ADR-ARCH-001` | 采用六边形架构，Domain 与 Application 不依赖 UI、Tauri、Python、数据库或 Spine | 确保推理、存储和导出可替换 | 在 React Store 、Tauri Command 或 Python 脚本中直接维护业务真值 |
| `F2S-ADR-ARCH-002` | Rust 核心是项目、工作流和文件写入的唯一权威 | 避免多进程并发写入和 schema 漂移 | Python Worker 直接修改项目文件或数据库 |
| `F2S-ADR-ARCH-003` | Python Worker 是可选、近似无状态的推理运行包 | 将 PSF/CUDA/模型权重的许可和体积与桌面核心分离 | 将 Python 解释器和全部模型强制捆绑到核心安装包 |
| `F2S-ADR-ARCH-004` | 内部渲染仅使用自有 Rig IR，Spine 4.2 JSON 是版本化边界输出 | 保持引擎中立并遵守 Runtime 许可约束 | 从官方 Runtime 复制解析或运行逻辑 |
| `F2S-ADR-ARCH-005` | 项目目录为事实来源，redb 只是可重建索引和任务库 | 项目可搬移、可人工取证，清缓存不丢源素材 | 将源图、Rig IR 或唯一的审批状态只存入索引库 |

## 4. 逻辑分层与组件

```text
├─ Presentation
│  ├─ React Shell / Forms / Panels
│  └─ PixiJS Editor Kernel
├─ Delivery
│  └─ Tauri Commands / Channels / Capability Policy
├─ Application
│  ├─ Use Cases / Command Bus / Query Services
│  ├─ Workflow Coordinator
│  └─ Job Scheduler
├─ Domain
│  ├─ Project / Specs / Asset Planning
│  ├─ Layer / Rig IR / Animation / Approval
│  └─ Validation / Quality Rules
└─ Infrastructure Adapters
   ├─ Project Repository / redb Index
   ├─ Local Worker / Private Remote GPU
   ├─ PSD / PNG / Spine42 JSON
   └─ User-owned Spine.com Process Adapter
```

| 组件 ID | 组件 | 职责 | 明确非职责 |
| --- | --- | --- | --- |
| `F2S-CMP-001` | Desktop UI | 展示、用户意图收集、局部交互状态 | 直接写项目、调模型、调 Spine CLI |
| `F2S-CMP-002` | Editor Kernel | 渲染 Rig IR 快照、命中测试、工具交互、代理纹理管理 | 提供持久化或最终 Spine 语义 |
| `F2S-CMP-003` | Tauri Delivery | 将有限命令和 Channel 暴露给指定 WebView | 向前端暴露任意 shell、fs 或 HTTP |
| `F2S-CMP-004` | Application Core | 用例编排、事务边界、授权、幂等和错误映射 | 包含具体文件、GPU 或 WebView API |
| `F2S-CMP-005` | Domain Core | 不变量、值对象、聚合、状态迁移、质量规则 | 依赖框架和外部进程 |
| `F2S-CMP-006` | Project Store | 原子保存、锁、迁移、内容寻址 Artifact | 解析用户动作语义 |
| `F2S-CMP-007` | Inference Gateway | 能力探测、适配本地/私有远程、取消和结果校验 | 自动越过人工门禁 |
| `F2S-CMP-008` | Export Gateway | 将已批准 Rig IR 转换为 PSD/PNG/Spine 4.2.43 JSON 候选产物 | 直接生成 `.spine`、捆绑 Runtime |

## 5. 核心端口

### 5.1 用例端口 `F2S-IFC-001`

```rust
trait CommandHandler<C> {
    type Output;
    fn handle(&self, command: C, ctx: CommandContext)
        -> Result<Self::Output, AppError>;
}

struct CommandContext {
    project_id: ProjectId,
    expected_revision: ProjectRevision,
    actor: LocalUser,
    idempotency_key: IdempotencyKey,
}
```

所有持久化修改必须携带 `expected_revision`。发生版本冲突时返回可恢复错误，不得以后到写入覆盖先到写入。

### 5.2 项目持久化用例端口 `F2S-IFC-002`

```rust
trait ProjectPersistencePort {
    fn inspect(&self, path: &ProjectPath) -> Result<ProjectInspection, AppError>;
    fn open(&self, path: &ProjectPath, mode: OpenMode)
        -> Result<ProjectSnapshot, AppError>;
    fn commit(&self, command: PersistProjectCommand)
        -> Result<ProjectRevision, AppError>;
    fn autosave(&self, command: PersistProjectCommand)
        -> Result<AutosaveRevision, AppError>;
    fn close(&self, project_id: ProjectId) -> Result<(), AppError>;
}
```

`F2S-IFC-002` 是 Application 使用的端口，不再声明第二个名为 `ProjectRepository` 的基础设施 trait。唯一存储 Repository 是 `F2S-DOC-STORE-001` 的 `F2S-IFC-401 ProjectRepository`，由 `ProjectRepositoryAdapter` 完成映射：

| `ProjectPersistencePort` | `F2S-IFC-401` / 其他唯一 owner | 说明 |
| --- | --- | --- |
| `inspect` | `ProjectRepository.inspect` | 只读检查，不迁移、不恢复 |
| `open` | `ProjectRepository.open` | `OpenMode` 与单写者锁由存储层实施 |
| `commit` | `ProjectRepository.commit` | Adapter 将用例命令转换为带 expected revision 的 `ProjectTransaction` |
| `autosave` | `ProjectRepository.autosave` | 与正式 commit 使用同一 schema，但不创建审批 |
| `close` | `ProjectRepository.close` | 释放会话与写锁 |
| 恢复 | `F2S-DOC-RECOVERY-001` 的恢复用例 | 不塞回 Repository 形成隐藏副作用 |
| 索引重建 | `F2S-IFC-403 IndexStore.rebuild` | 索引是可重建投影 |

`commit` 同时提交领域快照、新 Artifact 引用和审计事件。索引更新失败不得使已提交项目变为无法打开，应标记为待重建。

### 5.3 推理端口 `F2S-IFC-003`

```rust
trait InferenceProvider {
    fn capabilities(&self) -> Result<CapabilityManifest, InferenceError>;
    fn submit(&self, request: InferenceRequest) -> Result<JobHandle, InferenceError>;
    fn cancel(&self, job_id: JobId) -> Result<CancelReceipt, InferenceError>;
    fn verify_result(&self, result: WorkerResult)
        -> Result<JobOutputArtifact, InferenceError>;
}
```

实现只能是 `ManualProvider`、`LocalOrtProvider`、`LocalPythonProvider` 或 `PrivateRemoteProvider`。不存在通用公有云 Provider。

`F2S-IFC-003 InferenceProvider` 是 Application 层的稳定端口；`F2S-DOC-IPC-001` 的 `ComputeProvider`/`ArtifactGateway` 是 Delivery/Infrastructure 侧唯一协议接口。映射固定如下：

| Application 调用 | 12 号接口/适配器 | 边界结果 |
| --- | --- | --- |
| `capabilities` | `F2S-IFC-WORKER-005` capability handshake 或本地静态 manifest | 只产生能力投影，不启用未声明能力 |
| `submit` | `F2S-IFC-WORKER-001`、`F2S-IFC-WORKER-002`、`F2S-IFC-WORKER-003` 或 `F2S-IFC-RGPU-001 ComputeProvider` | 返回持久 JobHandle |
| `cancel` | `F2S-IFC-WORKER-006` | 只提交取消意图，终态由 Rust 仲裁 |
| `verify_result` | `F2S-IFC-WORKER-004 ArtifactGateway` | 产生已验证并持久到 CAS、但尚未绑定领域 candidate 的 `JobOutputArtifact` |

`ManualProvider`、`LocalOrtProvider`、`LocalPythonProvider`、`PrivateRemoteProvider` 分别映射到 `F2S-IFC-WORKER-001`、`F2S-IFC-WORKER-002`、`F2S-IFC-WORKER-003`、`F2S-IFC-RGPU-001`。协议 DTO 必须经适配器转换，不能直接实现领域对象。

### 5.4 导出端口 `F2S-IFC-004`

```rust
trait ExportAdapter {
    fn target(&self) -> ExportTarget;
    fn preflight(&self, snapshot: &ApprovedProject) -> ValidationReport;
    fn stage(&self, request: ExportRequest, temp: &StagingDir)
        -> Result<ExportManifest, ExportError>;
    fn verify(&self, manifest: &ExportManifest) -> Result<VerificationReport, ExportError>;
}
```

导出必须先写 staging 目录，结构、哈希和可选官方 CLI 往返验证通过后才原子提升到 `exports/`。

## 6. 依赖规则

1. `Domain` 只依赖 Rust 标准库和已审计的通用值类库。
2. `Application` 依赖 Domain 和 Port trait，不依赖 Adapter。
3. `Infrastructure` 实现 Port，可依赖 Application/Domain，不得反向暴露具体类型。
4. `Tauri Command` 只做 DTO 校验、用例调用和错误映射。
5. TypeScript 与 Python 的公共 DTO 由版本化 JSON Schema 生成，不手工保持三份同名类型。
6. React Store 只存页面、选择、相机和尚未提交的编辑状态。
7. Pixi 不读写项目文件，只消费 `RenderSnapshot` 并产生编辑意图。
8. Python Worker 不打开项目根目录，只能访问 Rust 为单个 Job 创建的受控目录。
9. 所有外部进程通过参数数组启动，禁止 shell 字符串拼接。

## 8. 设计模式的使用边界

| 模式 | 用于 | 不用于 |
| --- | --- | --- |
| Aggregate + Repository | Project、AssetPlan、Rig、Approval 一致性边界 | 把所有项目数据塞入单一巨型聚合 |
| Command | 可撤销编辑、审批和持久化修改 | 每一帧相机平移和鼠标 hover |
| Strategy | 推理档位、Worker Provider、动作表现模式 | 用于掩盖不兼容的数据模型 |
| Adapter | PSD、Spine42 JSON、Spine.com、私有远程 | 使外部格式反向污染 Rig IR |
| State Machine | 通用 Job 生命周期 | 整个 Production Assist 的巨型不可维护枚举 |
| Saga/Workflow | 跨阶段产物、人工门禁和补偿 | 代替单次原子保存 |
| Specification | 导出前、审批前和许可白名单规则 | 隐藏不可解释的 AI 分数 |
| Content-addressed Artifact | 不可变图片、mask、预览和导出中间件 | 快速变化的相机或面板状态 |

## 9. 错误分类和补偿

| 类别 | 示例 | 系统响应 | 数据保证 |
| --- | --- | --- | --- |
| Validation | schema 无效、状态迁移非法 | 拒绝命令并指向字段/规则 | 不写入 |
| Conflict | `expected_revision` 落后 | 刷新快照，让用户重放或放弃 | 不覆盖 |
| Transient | Worker 启动慢、私有端点暂时不可用 | 有上限的退避重试，显示原因 | 已批准产物不变 |
| Resource | OOM、磁盘不足、纹理超限 | 停止阶段，给出降级选项 | 不静默降质 |
| Integrity | Artifact 哈希不符、迁移中断 | 隔离结果，运行恢复报告 | 保留原版本 |
| External tool | Spine.com 缺失、超时、版本不符 | 候选输出可保留，官方验证标为未执行/失败 | 不伪造通过 |
| Cancelled | 用户取消长任务 | 发送取消，超时后终止 Worker | 临时产物可清，已提交产物不清 |

## 10. 安全与隐私边界

- Tauri Capability 按窗口最小授权；不使用全局 shell/fs/http 权限。
- 所有任意路径均在 Rust 解析和规范化，必须落在已打开项目或 Job sandbox 中。
- WebView CSP 默认 `connect-src 'self'`；私有端点也不由 WebView 访问。
- 无遥测、无广告 SDK、无默认更新检查；后续自更新必须在设置中明示并使用签名元数据。
- 日志不记录图片像素、完整提示词、凭据、Spine 激活码或不必要的绝对路径。
- 模型插件不允许动态执行未签名代码；仅允许已审计 manifest 所指向的本地 Worker 能力。

## 11. 性能与可用性预算

| 项目 | 设计预算 | 超限处理 |
| --- | --- | --- |
| 普通 UI 交互 | P95 输入反馈不高于 100ms | 查询异步化，避免全库序列化 |
| Pixi 预览 | 代理分辨率下目标 60 FPS，低档最低可接受 30 FPS | 降低预览级别，不改源产物 |
| 项目保存 | 普通元数据修改 P95 不高于 500ms | 延后索引重建，事实文件优先 |
| 崩溃恢复 | 不丢失最后一次成功 commit，未提交操作有恢复报告 | 进入只读恢复模式 |
| GPU 任务 | 同一 Worker 默认一个 GPU Job | 排队而非超分配显存 |

上述是验收预算而非已证明能力，必须在目标 Windows 硬件上生成性能证据。

## 12. 测试与架构门禁

| 测试 ID | 验证 | 通过条件 |
| --- | --- | --- |
| `F2S-TST-060` | 依赖方向检查 | Domain/Application 无 Tauri、Pixi、Python、redb、Spine 依赖 |
| `F2S-TST-061` | 适配器替换 | ManualProvider 与假 Worker 可通过同一用例套件 |
| `F2S-TST-062` | 前端最小权限 | 从 WebView 无法访问未授权文件、任意 shell 和公网 |
| `F2S-TST-063` | 并发修改冲突 | 落后 revision 必然失败且不覆盖新数据 |
| `F2S-TST-064` | Worker 终止隔离 | 杀死 Worker 不使桌面崩溃，Job 只能进入精确机器状态 `interrupted` 或 `failed` |
| `F2S-TST-065` | 不捆绑 Spine Runtime | 源码、锁文件、安装包和 SBOM 都不含官方 Runtime |
| `F2S-TST-066` | 端到端纵切 | 导入图片→人工 AssetSpec→分层→Rig IR→Spine 4.2.43 JSON 可重复运行 |

架构门禁失败时，不得通过在 UI 增加特判或复制领域逻辑规避，必须回到 canonical 接口或聚合修改。

## 13. 许可与外部组件边界

发布政策只引用14号文档的`F2S-LIC-POLICY-001`，本架构不复制或扩展SPDX列表。每个发布组件仍需清单证据；policy外、禁止或未知许可不进入发布包。

许可必须分类审计：

1. 随桌面核心发布的库；
2. 随可选 AI Runtime Pack 发布的 Python 库和本地模型；
3. 操作系统或硬件的专有前置，如 WebView2、GPU Driver/CUDA；
4. 用户自有的 Spine Professional/Enterprise 和 `Spine.com`；
5. 仅在构建期使用且不进入产物的工具。

任何一类都需要版本、来源、SPDX/自定义许可、哈希和分发方式。“代码仓库是 Apache-2.0”不能自动证明模型权重和训练数据可商用。

## 14. 架构变更流程

以下任一改动必须新增 ADR，而不能直接在代码中生效：

- 增加新的目标引擎或新 Spine 版本；
- 将 Python Worker 改成核心必选组件；
- 开放任意公网、插件或脚本执行；
- 将项目事实从目录文件迁往数据库；
- 捆绑 Spine Runtime、WebView2 Fixed Runtime、CUDA 或新模型权重；
- 破坏 schema、Worker Protocol 或 Rig IR 向后兼容性。

变更 ADR 必须列出被影响的 F2S ID、数据迁移、许可差异、回滚方案和重新执行的测试。
