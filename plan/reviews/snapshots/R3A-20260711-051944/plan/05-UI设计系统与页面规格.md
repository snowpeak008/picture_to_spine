# UI 设计系统与页面规格

## 1. 设计目标

界面服务于长时间、高密度的 2D 生产工作。它必须同时满足：

- 画布和时间轴拥有最大可用空间；
- 候选、工作版本、批准版本和失效版本不会混淆；
- AI 派生像素、诊断和人工审批始终有可见来源；
- 同一对象在树、画布、时间轴、检查器和问题中心保持同步选择；
- 高 DPI、多显示器和中文输入法下行为稳定；
- 核心流程可通过键盘完成，状态不只依赖颜色；
- 长任务不冻结界面，也不通过弹窗阻塞无关工作。

首版采用深色生产主题。颜色结构预留浅色主题变量，但浅色主题不作为 P0，不能因此产生业务层硬编码。

## 2. 设计语言

### 2.1 视觉基调

界面应克制、技术化且可读。角色图片是视觉主角，应用自身避免大面积高饱和装饰、玻璃拟态和持续动画。只有当前工具、关键帧、审批和阻塞错误使用高强调。

### 2.2 密度等级

- 默认 `comfortable-dense`：适合 1440p/1080p 桌面；
- 表格/树行高 28 px；
- 普通控件高 32 px；
- 主要操作高 36 px；
- 触摸不是首版目标，但可点击区域不得小于 28×28 px；
- 用户可将面板 UI 缩放设为 90%/100%/110%/125%，画布缩放独立。

### 2.3 图标与文字

图标只用于加速识别，不能替代未知状态、审批和破坏性操作的文字。所有仅图标按钮有 tooltip、可访问名称和可见焦点。

## 3. Design Tokens — `F2S-UI-TOKEN-001`

### 3.1 颜色

| Token | 值 | 用途 |
| --- | --- | --- |
| `color.bg.app` | `#101318` | 应用背景 |
| `color.bg.surface` | `#171B22` | 面板、树、检查器 |
| `color.bg.elevated` | `#202631` | 菜单、弹层、选中组 |
| `color.bg.canvas` | `#0B0E12` | 画布默认背景 |
| `color.border.default` | `#65758D` | 分隔、输入框；对 `surface/elevated` 中较差组合仍达到至少 3:1 |
| `color.border.focus` | `#66B3FF` | 键盘焦点 |
| `color.text.primary` | `#F2F5F8` | 主文字 |
| `color.text.secondary` | `#AAB4C2` | 辅助文字 |
| `color.text.disabled` | `#8E99A8` | 禁用文字；不得低于正文背景上的 4.5:1 对比 |
| `color.accent` | `#66B3FF` | 当前工具、链接、主选择 |
| `color.success` | `#4FD08B` | 已通过，同时使用 check 图标 |
| `color.warning` | `#F5C451` | 警告，同时使用 triangle 图标 |
| `color.danger` | `#FF6B73` | 错误/破坏，同时使用 octagon 图标 |
| `color.stale` | `#B58CFF` | 已失效，同时使用 history 图标 |
| `color.aiDerived` | `#20C9B5` | AI 新像素/候选，使用 sparkle/hatched overlay |
| `color.userApproved` | `#4FD08B` | 人工批准，不与 success 文字单独混用 |
| `color.timeline.playhead` | `#FF5DA2` | 播放头 |

所有常规文字和控件状态必须通过 WCAG 2.2 AA 对比验证。`color.border.default` 还须在 `surface/elevated/input` 相邻组合中通过非文本控件 3:1 边界检查；disabled 文本仍按 4.5:1 项目内硬门执行。颜色值变化只允许修改 token，不允许页面使用散落十六进制值。

### 3.2 字体

- UI：Windows 系统 `Segoe UI`、`Microsoft YaHei UI` 回退链，不随安装包重新分发字体；
- 等宽：系统 `Cascadia Mono`/`Consolas` 回退；
- 字号：12（元信息）、13（树/表格）、14（正文/控件）、16（区块标题）、20（页面标题）、24（空状态标题）；
- 行高：正文 1.45，紧凑表格 1.3；
- 数值字段使用 tabular numbers；
- 路径、ID、hash、帧号使用等宽字体。

### 3.3 间距、圆角与层级

- 4 px 基础网格：`space.1=4`、`2=8`、`3=12`、`4=16`、`6=24`、`8=32`；
- 圆角：输入/按钮 4 px，面板 6 px，模态 8 px；
- 画布和时间轴的选择轮廓为 1 px + 高对比外沿；
- 阴影只用于菜单、tooltip、拖拽 ghost 和模态；
- 分隔主要依靠层级色与 1 px border，避免重复卡片套卡片。

### 3.4 动效

- hover/focus 80–120 ms；
- 面板展开 120–160 ms；
- 禁止对画布对象位置、关键帧和审批状态做缓动过渡；
- 尊重 Windows “减少动画”设置；
- 任务进度未知时使用低干扰循环条，不让整页闪烁。

## 4. 应用布局 — `F2S-UI-LAYOUT-001`

### 4.1 窗口基线

- 最小支持窗口：1280×720；
- 推荐：1920×1080 或更高；
- 低于最小尺寸时允许窗口存在，但显示“空间不足”提示并启用面板自动折叠；
- 支持 100%、125%、150%、200% DPI；
- 主窗口位置必须钳制在当前可用显示器范围，拔掉副屏后不可丢失；
- 全屏、最大化、普通窗口状态按项目外全局偏好保存。

### 4.2 工作区尺寸

| 区域 | 默认 | 范围 |
| --- | --- | --- |
| 标题/命令栏 | 44 px 高 | 固定 |
| 阶段导航 | 216 px 宽 | 168–300，可折叠到 48 |
| 对象树/工具架 | 240 px 宽 | 180–420 |
| 检查器 | 320 px 宽 | 260–560 |
| 底部抽屉 | 240 px 高 | 160–60% 窗高，可收起至 28 |
| 状态栏 | 24 px 高 | 固定 |
| 中央画布 | 剩余空间 | 不低于 480×320 |

所有分隔条支持鼠标拖动、键盘微调和双击恢复默认。布局按页面类型保存，例如 Layer、Rig、Animation 各自保存。

### 4.3 标题栏

从左到右：

1. 应用/项目菜单；
2. 项目名和当前 revision；
3. 保存状态：`已保存/未保存/保存中/恢复草稿/下游失效`；
4. 当前页面/对象简要路径；
5. 计算位置：`离线本地/本地 GPU/私有远程`；
6. 任务按钮和活动数；
7. 问题按钮及错误/警告数；
8. 帮助、窗口控件。

标题栏不得显示笼统的“100%完成”；概览通过门禁列表表达真实状态。

## 5. 核心组件 — `F2S-UI-COMP-001`

### 5.1 状态徽标

`StatusBadge` 接收 `F2S-DOC-UX-001` 定义的工作流阶段、对象生命周期、新鲜度、阻塞或导出验证枚举，并明确标注所属维度；输出图标、短文字、颜色和 tooltip。禁止调用方传任意颜色制造新状态，也禁止用单个 badge 覆盖多个维度。

示例：

- ✓ 已批准；
- ◇ 候选；
- ! 需审核；
- ↺ 已失效；
- ⛔ 阻塞；
- ? 未验证。

### 5.2 来源徽标

`ProvenanceChip` 区分：

- 用户导入；
- 用户手绘；
- 规则建议；
- 本地 AI；
- 私有远程 AI；
- 外部提示词回导；
- 迁移产生。

点击后打开来源抽屉，显示源 hash、遮罩、任务和审批，不在普通 tooltip 暴露敏感路径。

### 5.3 门禁卡

`GateCard` 包含：

- 门名称和状态；
- 已检查/总项；
- 阻塞问题；
- 最近批准 revision；
- 上游变化；
- 主操作“检查并审批”或“重新验证”。

门禁卡禁止只有绿色百分比。审批命令只能在 Gate Review 对话框中发出。

### 5.4 对象树

支持分组、筛选、搜索、锁定、可见、展开、状态、拖放和多选。拖放过程中显示合法目标；重父级、draw order 或语义改变在 drop 前预览结果。大树使用虚拟化，但键盘导航和屏幕阅读顺序保持稳定。

### 5.5 属性检查器

属性按“常用—变换—结构—来源—诊断—高级”分组：

- 有效数值提交后才进入 Command；
- 拖动 scrubber 时预览，释放时形成一个撤销命令；
- 混合多选值显示 `—`；
- 单位始终可见；
- 无效值保留输入并显示错误，不把模型写成 NaN；
- 改变结构或上游真值时显示影响图标。

### 5.6 问题行

每行含严重度、类别、规则/需求 ID、对象、动作/帧、摘要、状态、`waiverPolicy` 和“前往”。展开显示证据帧、阈值、建议与允许动作。组件契约必须要求 `waiverPolicy: forbidden | user_acknowledgement | governed`：`forbidden` 不渲染接受按钮；`user_acknowledgement` 只显示“确认艺术建议”并要求备注；`governed` 只为P2显示“申请延期”，进入 `F2S-REL-WAIVER-001`。P0/P1和安全、隐私、许可、用户数据完整性/恢复类别必须由领域层传入 `forbidden`，UI不得覆盖。错误不能只以 toast 出现。

### 5.7 任务行

显示执行位置、阶段、进度、资源、取消、未绑定输出和注册状态。Job `succeeded` 后主按钮是“注册为候选并审阅”，它提交带 expected revision 的独立命令；注册冲突时保留输出并提供刷新重试。只有注册成功后才显示对象 `candidate`，任何情况下都不是“应用”或“批准”。

### 5.8 比较器

支持：

- A/B 切换；
- 可拖动分割线；
- alpha 叠加；
- 差异热图；
- 同步缩放/平移；
- 帧同步播放。

比较标题明确写出 `批准 r12` 与 `工作 r16`，避免“之前/现在”含义随选择变化。

### 5.9 提示与确认

- Toast 仅用于保存成功、复制成功等短暂非阻塞反馈；
- 可修复错误进入 inline error 或问题中心；
- 破坏性/失效操作使用影响确认；
- 系统级失败使用非阻塞 banner，并提供详情；
- 模态只用于审批、不可逆发布、凭据和必须立即选择的恢复。

## 6. 画布系统 — `F2S-UI-CANVAS-001`

### 6.1 坐标与显示

- 画布坐标、项目坐标和屏幕坐标必须明确转换；
- DPI 缩放不改变项目像素和 pivot；
- 状态栏显示光标项目坐标、选中对象局部坐标和当前缩放；
- 画布缩放范围建议 5%–3200%，靠近像素级时提供 pixel grid；
- 支持透明棋盘、纯色、暗/亮和用户背景；
- 图像采样可切换 nearest/linear，仅影响预览。

### 6.2 工具状态

画布一次只有一个主工具，辅助键临时覆盖：

| 工具 | 主要操作 |
| --- | --- |
| Select/Transform | 选择、框选、移动、旋转、缩放 |
| Pan/Zoom | 平移、缩放、适合选择/画布 |
| Mask Brush/Eraser | 蒙版绘制与擦除 |
| Lasso/Fill | 区域选择和填充 |
| Bone Create/Edit | 创建和编辑骨骼 |
| Mesh Vertex/Edge | 顶点、边和三角形 |
| Weight Paint | 权重绘制、平滑、归一 |
| Pivot/Socket | pivot、地面、握持和 socket |
| Pose | 在动画时间写入变换关键帧 |
| Review Marker | 审核批注，不修改动画 |

切换页面时工具映射到安全默认。进行未提交的套索、骨骼创建或权重 stroke 时切页，先提交或取消当前手势，不能留下半命令。

### 6.3 选择同步

- 单击画布对象同步对象树和检查器；
- Alt/循环选择处理重叠对象；
- 从树选择不可见对象时显示边界但不强制改可见性；
- 锁定对象不响应普通点击，但可从树检查；
- 选择动作专用附件时，动作上下文自动切换需确认，避免误编辑错误 clip。

### 6.4 叠加层

叠加层按固定顺序绘制：

1. 背景和地面；
2. 角色；
3. AI 派生像素覆盖；
4. mesh/weight；
5. bone/constraint；
6. pivot/socket/contact；
7. 诊断热图；
8. 选择与工具柄；
9. 审核批注。

用户可逐项关闭；阻塞诊断跳转时临时开启相关叠加并显示“已临时开启”。

### 6.5 蒙版体验

- 画笔游标显示实际项目像素直径；
- 压感仅在检测到设备时启用；
- stroke 结束形成单一命令；
- 大笔刷预览节流但最终蒙版不能丢采样点；
- 显示前景、背景和未知区三态；
- AI 补全遮罩外写入被视为错误并高亮；
- 原图图层始终只读。

### 6.6 网格与权重

- 顶点、边、三角形各有独立选择样式；
- 非法三角形立即显示错误但不破坏上一有效 mesh；
- 权重颜色配合骨骼编号/名称，不能只靠渐变；
- 影响骨列表显示精确数值、总和和归一提示；
- “自动权重”产生候选层，可 A/B 比较后接受；
- 超出硬预算在提交前阻止，并说明受影响导出。

## 7. 时间轴 — `F2S-UI-TIMELINE-001`

### 7.1 结构

```text
动作栏：动作选择 / 状态 / 时长 / FPS 显示 / 循环 / 播放
阶段栏：anticipation / active / recovery 等 MotionSpec 区段
标尺：秒与帧双显示
轨道树：bone / slot / deform / constraint / event / sequence
dope sheet：关键帧
曲线编辑器：与 dope sheet 切换或分屏
底部：当前时间、选中关键帧、吸附和范围
```

### 7.2 关键帧语言

- 不同轨道类型用形状区分，颜色辅助；
- 已选中、存在警告、stale 和被锁定分别有轮廓/图标；
- stepped、linear、bezier 在关键帧连线上可见；
- 多选移动显示 delta，并在越界/覆盖时预览冲突；
- 事件和命中候选位于专用 lane，不与普通骨骼关键帧混淆；
- 用户批准的 hit 标记具有锁形/签章图标，修改影响时变为 stale。

### 7.3 播放

- Space 播放/暂停，逗号/句号逐帧；
- 支持全动作、工作范围和选区循环；
- 速度 0.1×–2×，审核默认 1×；
- 掉帧时显示实际 FPS，不通过跳过关键审核帧伪装流畅；
- onion skin 支持前后 1–5 帧、低透明度和轮廓模式；
- 参考姿势图可按当前帧锁定和调透明度。

### 7.4 阶段和游戏标记

MotionSpec 阶段以可调区间显示。模板建议使用虚线和“候选”标签；人工批准后为实线。移动关键帧跨过阶段边界时提示，不自动改变玩法标记。

## 8. 页面规格

### 8.1 首页 — `F2S-SCR-HOME-001`

**布局**

- 左侧：新建、打开、恢复；
- 中央：最近项目列表，显示路径、最后时间、兼容/缺失状态；
- 右侧：应用版本、离线隐私说明、Spine 4.2.43 检测摘要；
- 底部：设置、日志、许可证。

**状态**

- 无最近项目：提供“新建”和“打开”说明；
- 路径缺失：灰显但可定位/移除；
- 检测到恢复：顶部高优先 banner，不能被最近项目卡遮挡；
- 更新提示不得包含自动遥测，首版可完全不实现在线更新。
- Spine 区域固定显示 `not_configured`、`path_missing`、`version_mismatch` 或 `4_2_43_detected_unverified`；在合法本机往返证据产生前持续显示“实验/未验证”徽标，不能把离线版本探测显示为兼容通过。

### 8.2 新建项目 — `F2S-SCR-NEW-001`

分三组而非长向导：

1. 名称与目录；
2. 内容基线：侧视类人、`primary_weapon`、十个动作（只读说明）；
3. 画布/计算初值。

路径检查实时但防抖；创建按钮只有所有必填通过时启用。显示将创建的目录，不静默使用系统盘。

### 8.3 项目概览 — `F2S-SCR-OVERVIEW-001`

**顶部**：项目状态、当前 revision、下一推荐动作。

**主区**

- 九个阶段 GateCard；
- 十个动作状态矩阵；
- 素材覆盖：计划/导入/批准/缺口；
- 阻塞问题和 stale 对象；
- 最近任务与导出；
- 存储、内存/显存预算。

概览不使用单一圆环百分比。每个卡片可跳转至相应对象。

### 8.4 输入与母版 — `F2S-SCR-IMPORT-001`

**布局**

- 左：输入版本/文件列表；
- 中：原图、规范化、差异比较画布；
- 右：文件属性、朝向、地面、原点、包围盒、身份锚点；
- 底：导入检查和母版门禁。

**主要命令**：导入、建立候选、比较、提交审批、批准/退回。

原始输入旁始终显示只读锁；任何编辑目标必须注明“工作副本”。

导入区只把 PNG、JPEG、WebP 显示为 P0 可选格式。PSD 出现在拖放或选择结果中时，页面必须保留现有项目状态、显示“V1 不支持 PSD 输入”和转换说明，不创建半成品候选。

### 8.5 规格编辑器 — `F2S-SCR-SPEC-001`

**左栏**：Style、Character、Weapon、Runtime、十个 MotionSpec。

**中央**：结构化表单和母版/关键姿势参考。

**右栏**：字段来源、置信度、验证、影响和审批。

必填但未决字段使用 `UNRESOLVED`，不能保存成看似合法的空字符串。WeaponSpec 未完成时三个 attack 项显示阻塞链。

### 8.6 提示词包 — `F2S-SCR-PROMPT-001`

**布局**

- 左：按 global/action/asset 分组的 PromptItem 树；
- 中：已解析提示词、模板/解析结果 diff、正向/负向 tabs；
- 右：引用的 Spec revision、母版 hash、未决占位符、关联 AssetSpec；
- 底：外部生成图片回导区。

**主要命令**：重新解析、复制当前、导出包、回导图片、关联候选。

复制时 toast 显示 PromptItem ID；若有未决占位符，复制文本首行自动加开发草案警告，且生产导出按钮禁用。

回导表单显示 provenance：内部 AI 任务必须完整且只读展示 Provider、模型/算法版本、参数、输入/输出哈希和任务 ID；外部图片无法提供生成信息时，用户必须选择 `unknown_external` 并填写来源声明。来源未知徽标在候选、审批和导出报告中持续可见。

页面不出现“生成图片”按钮，避免暗示内置第三方生图。

### 8.7 素材计划 — `F2S-SCR-ASSETPLAN-001`

提供三种视图：

- 动作 × 素材矩阵；
- AssetSpec 表；
- 闭环问题/缺图队列。

选择单元格显示表现策略、来源规则和证据帧。用户覆盖策略必须从原因列表选择并可补备注。孤儿素材、重复候选和新视角超范围有显著标记。

### 8.8 分层与补全 — `F2S-SCR-LAYER-001`

**布局**

- 左：语义图层树；
- 中：双/三视图画布，可切母版、mask、层、重组、差异；
- 右：语义、父骨建议、pivot、重叠、来源；
- 底：补全任务、逐层审批和诊断。

工具栏显示当前操作目标：`蒙版`、`图像像素` 或 `属性`。AI 新像素默认用斜线覆盖，用户可按住临时查看原始结果。

### 8.9 Rig 工作台 — `F2S-SCR-RIG-001`

**左**：bone/slot/attachment/constraint 对象树和模式 tab。

**中**：Rig 画布。

**右**：变换、结构、mesh/weight、导出映射。

**底**：临时姿势/动作 mini timeline 与诊断。

工具模式变化时标题附近显示当前模式和快捷键。结构删除、重父级和 rest pose 改动显示影响预览。

### 8.10 动画工作台 — `F2S-SCR-ANIM-001`

**顶部**：十动作 selector，始终分别显示对象生命周期（缺失/草稿/需审核/批准）与新鲜度（current/stale）；blocked 使用独立问题标记。

**中央**：画布与参考比较。

**右**：当前对象/关键帧/曲线属性和 MotionSpec。

**底**：可扩展时间轴。

攻击动作显示阶段条、武器接触和命中候选；这些轨道的模板建议和用户批准必须有不同样式。

### 8.11 审核页 — `F2S-SCR-REVIEW-001`

页面默认只读，避免审核过程中意外改动画：

- 左：十动作审核队列；
- 中：当前/批准版本同步预览；
- 下：关键姿势胶片条、阶段和事件；
- 右：检查表、诊断、批注和批准/退回。

需要修改时点击“回到动画编辑”，保存返回位置。attack 审核必须逐个显示 contact 证据，不允许把三个命中帧默认全选。

### 8.12 导出页 — `F2S-SCR-EXPORT-001`

分为四步但保持同页：

1. Preflight；
2. 产物与目录；
3. Spine CLI 检测与可选验证；
4. 结果和报告。

错误行可以跳转；警告动作严格服从 IssueRow 的 `waiverPolicy`，确认或P2延期均进入报告但不把失败要求改成通过。产物写入使用 Job 状态，成功提交后工作流阶段为 `exported`；Spine 检查单独显示 `exported_unverified/verification_running/spine_verified/incompatible`。总状态不得用一个绿色“成功”掩盖未验证 Spine。PSD 项必须区分“P0 最小可重建契约通过”和“高级特性已降级”。

CLI Adapter 初始状态为 `experimental/unverified`。默认“检测版本”只能执行离线 `--version` 探测并显示路径、实际patch、时间和脱敏结果；它不产生 `spine_verified`。任何可能执行 `--update 4.2.43` 或联网的动作必须由用户单独勾选目标、网络影响和许可责任后确认。导出页不得默认运行更新或转换；转换后仍须显示实际patch证明与语义比较结果。

### 8.13 设置 — `F2S-SCR-SETTINGS-001`

分类：

- 常规/语言/布局；
- 本地计算与资源预算；
- 私有远程 GPU；
- Spine 4.2.43 CLI；
- 项目保存与缓存；
- 日志与隐私；
- 许可证和实验功能。

Spine 设置卡必须同时展示：Adapter实验状态、配置路径、离线probe结果、目标4.2.43与实际patch差异、最后验证证据、联网动作默认关闭。版本不符时允许保存配置但禁用“标记已验证”；只有用户显式许可的命令可进入确认对话。

删除缓存前显示可重建项，永远不列入项目源资产。远程端点的“测试连接”说明不会上传项目图。

### 8.14 恢复页 — `F2S-SCR-RECOVERY-001`

三个候选以时间线/比较表显示：确认保存、自动保存、任务 checkpoint。每个显示完整性、对象差异和建议；默认选中“打开确认保存”，但不自动执行。损坏候选禁用恢复并提供隔离/诊断。

### 8.15 关于页 — `F2S-SCR-ABOUT-001`

关于页是可审计信息入口，不是单纯品牌弹窗：

- 产品显示名、SemVer、build number、Git commit、构建时间和目标架构；
- 项目 schema、Rig IR schema、Worker protocol 与 Spine 目标 `4.2.43`；
- 当前能力状态明确区分 VERIFIED、PARTIAL、UNVERIFIED，并可打开证据摘要；
- SBOM、`THIRD_PARTY_NOTICES`、Model BOM、许可政策和隐私说明可离线查看；
- 显示“未捆绑 Spine Editor/Runtime”“图片不发送第三方公有云”“私有远程 GPU 仅由用户显式启用”；
- 提供复制脱敏版本信息、打开本地日志目录和生成诊断清单的入口；
- 不显示或复制项目名、绝对路径、图片、完整提示词、凭据和 Spine 激活信息；
- 外部链接在交给系统浏览器前显示目标域名并要求确认，离线时页面核心信息仍完整。

## 9. AI 辅助交互规范

### 9.1 候选不等于结果

所有自动建议进入 Candidate Panel：

- 输入和使用的 Spec revision；
- 执行位置；
- 结果缩略图/结构；
- 置信度仅在有校准依据时显示；
- A/B 和差异；
- 接受、部分采用、拒绝；
- 接受后仍需领域审批。

### 9.2 AI 派生像素

- 默认以青绿色斜线覆盖；
- hover 显示来源 mask 和任务；
- 切换“仅 AI 区域”；
- 遮罩外变化显示红色错误；
- 用户手工修改后来源变为“AI 派生 + 用户修正”，不能抹去历史。

### 9.3 不确定性

系统使用具体语言：

- “模型未识别到远侧前臂，请用户标注”；
- “该姿势可能需要强透视附件”；
- “尚未运行检查”。

禁止含糊状态：“AI 已优化”“智能完成”“质量良好”。

## 10. 命令、撤销与保存反馈

UI 所有持久编辑通过 Command 边界：

- 文字/数值编辑：确认或失焦形成命令；
- 连续拖动：pointer down 到 pointer up 为一个命令；
- mask/weight stroke：一次 stroke 一个命令；
- 批量操作：一个复合命令，可展开查看子项；
- 外部候选接受：引用 blob 的命令，不直接覆盖；
- 审批：不可与内容编辑合并，是独立审计命令；
- 导出和任务启动不是可撤销编辑，但可取消并保留记录。

保存反馈：

- 标题栏即时显示 dirty；
- 保存中显示阶段，但不阻塞编辑时需创建下一 revision；
- 保存失败保留 dirty 和恢复草稿，提供重试/另存；
- 不使用只出现 2 秒的 toast 作为保存失败唯一通知。

## 11. 可访问性 — `F2S-UI-A11Y-001`

### 11.1 键盘

- Tab 顺序遵循标题栏 → 导航 → 页面工具 → 主区 → 检查器 → 底部抽屉；
- F6 在主要区域间循环；
- Escape 取消当前工具手势或关闭最上层弹层；
- Enter 提交普通表单，但审批对话框必须明确聚焦且不能因误回车直接批准；
- 菜单、树、表格、tab、splitter 和时间轴提供 ARIA/可访问角色与键盘行为。

### 11.2 视觉

- 正文和控件满足 AA 对比；
- 自动 contrast 测试必须遍历 token 在 app/surface/elevated/canvas/input、hover、focus、selected、disabled、error、warning、stale 组合中的文本 4.5:1 与非文本控件 3:1 门槛；截图抽查覆盖 100%/125%/150%/200% DPI；
- 焦点环宽至少 2 px 且不被裁切；
- 错误、警告、成功、stale 使用图标和文本；
- 画布 overlay 提供高对比/色觉缺陷预设；
- 缩放 200% 时关键流程可滚动到达，不要求横向与纵向同时精确操作。

### 11.3 画布替代信息

专业画布无法完全用屏幕阅读器操作，但对象树、属性、坐标、骨骼层级、关键帧表和诊断必须提供文本等价入口。核心项目管理、审批、导出和设置可不依赖画布完成。

## 12. 响应性与性能反馈

- 画布交互优先，不因后台缩略图、诊断或保存明显卡顿；
- 超过 100 ms 的操作显示局部 pending；超过 1 秒显示阶段；
- 不确定时长使用 activity 而伪造百分比；
- 树、表和时间轴采用虚拟化，但滚动位置、选择和可访问焦点不丢失；
- 高成本差异热图可降分辨率预览，最终审批前运行全分辨率；
- 内存/显存接近预算时状态栏先警告，并提供降级建议；
- GPU 设备丢失时画布尝试恢复或切软件渲染，用户工作状态不丢失。

## 13. UI 文案规范

- 主按钮使用动词 + 对象：“批准母版”“导出当前版本”“审阅候选”；
- 不使用“确定”承载不同危险程度；
- 禁用控件旁提供原因和可执行前置步骤；
- “删除”只用于可恢复移除；永久清理明确写“永久删除缓存副本”，且不能包含源资产；
- “验证”只用于实际运行检查，“兼容目标”不等于“已验证”；
- “私有远程 GPU”不得缩写成容易被误解为公共云的“云端”；
- “AI 提示词包”旁持续说明“本应用不发送或生成图片”。

## 15. UI 不合规清单

出现以下设计即判为不合规：

- 把 AI 候选生成完成显示为“已批准”；
- 只用颜色区分状态或权重；
- 用不可撤销自动覆盖替代候选比较；
- 在提示词页提供未获授权的第三方生图调用；
- 把 Spine 文件写出和 Spine 4.2.43 验证合成一个“成功”状态；
- 在用户未查看影响范围时使大量下游审批失效；
- 保存失败仅用短 toast；
- 隐藏原始输入与工作副本差别；
- 用百分比掩盖某一必需动作或审批缺失；
- 私有远程任务未显示上传对象；
- 缓存清理界面可能选择项目源资产；
- 在 125%/150%/200% DPI 下使用未经坐标转换的画布命中测试；
- 模态审批默认聚焦“批准”并允许误回车提交；
- 图标按钮无 tooltip、可访问名称或焦点状态。
