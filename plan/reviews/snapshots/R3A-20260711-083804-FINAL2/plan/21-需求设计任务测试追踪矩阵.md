# 需求—设计—任务—测试追踪矩阵

## 1. 用途与状态

本矩阵防止多份计划之间出现孤儿需求、无需求设计、无验收任务和无证据测试。当前revision已经为每个FR/NFR预留exact DEV和TST ID；`plan/devplan`必须按这些ID创建任务卡，禁止重新编号。原子计划完成后补入实际evidence path和执行状态，revision再次升级。

固定链路为：

```text
GOAL → UC → FR  → exact AC ───────────────────→ TST → DEV → EVD → RELEASE
          └→ NFR → canonical量化规范即验收合同 ─┘
FR/NFR/CON → ADR/CMP/IFC/ENT/SCR/ACT（设计实现边）
```

同一 ID 只允许一个 canonical owner。其他文件只能引用，不得重新定义不同语义。
