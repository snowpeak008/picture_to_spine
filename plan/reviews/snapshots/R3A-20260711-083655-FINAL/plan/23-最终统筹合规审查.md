## 3. 产品与范围门禁

- `F2S-COMP-SCOPE-001`：产品明确是闭源商业 Windows Production Assist，而非单图全自动承诺。
- `F2S-COMP-SCOPE-002`：只接收母版、关键帧图片和动作描述，不调用第三方生图API。
- `F2S-COMP-SCOPE-003`：V1仅二次元类人、横版侧视、单`primary_weapon`，固定十个clip。
- `F2S-COMP-SCOPE-004`：项目依用户授权完成五选一，V1 仅集成 Spine Editor 4.2.43；四类 Runtime/引擎集成均列为非目标。
- `F2S-COMP-SCOPE-004A`：内置Spine输出只允许JSON、透明attachment PNG和atlas packing/input manifest；`.atlas/.spine/.skel`只有用户本地合法Professional CLI 4.2.43可生成。
- `F2S-COMP-SCOPE-005`：母版、分层、Rig、关键姿势、命中帧、最终QA均有不可绕过的人工门。
- `F2S-COMP-SCOPE-006`：提示词包是正式输出，但外部生成质量不被软件虚假担保。

## 4. 架构与设计模式门禁

- Domain/Application不依赖Tauri、React、Pixi、Python、redb、HTTP或Spine。
- React不直接读写文件、联网或启动进程；Tauri capability最小授权。
- `project.json`当前指针、版本化manifest与其引用的不可变CAS共同构成项目事实源；Rig IR是Rig/动画领域事实，Pixi只持有瞬时渲染状态；redb只做可重建索引。
- Command支持审计与Undo/Redo；长任务由分层状态机/Saga编排；Adapter隔离Worker/PSD/Spine。
- Python Worker无权修改project manifest；二进制通过受控相对路径和hash传递。
- 不使用“设计模式”包装简单纯函数或制造无收益抽象。

## 5. 数据、状态与恢复门禁

- 原图和已批准artifact不可变，编辑产生新revision；
- Stage、Lifecycle、Freshness、Blockage与ExportVerification五维正交；Job另为执行维，`stale`只属于Freshness，历史版本由revision关系表达；
- target decision命令CAS project revision、ApprovalStreamRegistry及target entry；Approve另CAS current PriorityRegistry observation、WaiverRegistry→ActorRegistry/enrollment/trust+revocation bundle roots和clockEpoch/checkpoint双head。持续有效性绑定target、DependencySnapshot、stable PrioritySlice/inputs/outcome、DecisionPolicy、精确ack及static binding，并按current priority subjects、固定ActorAttestationPolicy、publisher签名且edition/expiry有效的OrganizationTrustEnrollment、waiver entry、同exact waiverId/proposal三credential签名、ActorTrust/Revocation状态和ClockEvidence动态验证；full registry root只作transport审计，无关entry不失效；
- ClockRecovery只能创建`oldEpoch+1`的epoch-boundary successor checkpoint：全项目sequence+1、prev artifact/domain双指旧epoch head，并与唯一RecoveryRecord old/new双head配对；prev-null、跳sequence、伪epoch或旧epochgrant复活均fail closed；
- release signer必须重算并双hash固定`ReleaseGateCoverageReportV1`，从current配置枚举全部required gate targets，逐项证明current ApprovalStreamRegistry中的active approve、精确ack及current waiver validation；缺gate/target、旧head、stale/inactive或未覆盖warning任一阻断签名；
- 保存算法只引用`F2S-DOC-STORE-001`的canonical事务和`F2S-FIX-FAULT-001`，本文不复制步骤顺序；
- 项目单写者锁，缓存可清除且不丢源数据；
- 崩溃、断电、磁盘满、Worker挂掉、OOM、远端断线、迁移中断有恢复或明确终止状态；
- 取消与完成竞态只能有一个最终状态；失败输出进入quarantine。

## 6. 安全与隐私门禁

- 默认网络关闭，无遥测，无自动诊断上传；
- 远程GPU仅用户自托管/局域网/企业私有端点，每项目显式启用；
- 上传前展示目标、文件、大小和数据分类；公网第三方默认拒绝；
- 密钥进入Windows安全存储，不进入项目、日志或诊断包；
- 文件签名/尺寸/路径/解码限制、压缩炸弹和路径遍历检查；
- 外部进程使用可执行路径与参数数组，不经过shell拼接；
- WebView CSP和capability最小化；前端无通用shell/HTTP能力；
- 日志默认不含图片、prompt全文、token、目录清单和绝对路径。
- 商业D2 Worker只允许`windows-appcontainer-v1`；restricted AppContainer、无network capability、专用ACL、Job Object和恶意探针缺一即禁用Provider并移除Worker Pack。
- Actor签名必须同时绑定固定SignatureAlgorithmRegistry：ECDSA P1363/low-S、RSA-PSS exact parameters及strict unpadded base64url；provider默认/宽松解码不可接受。有waiver的Stable签名前必须生成绑定本candidate/current roots/ClockEvidence及同signingAttemptId的OrganizationBundleFreshnessDeclaration，最终签名前年龄≤900秒，换attempt/重启/超时重签；它只证明操作者按流程刷新，不证明离线global latest，特权whole-state rollback和未导入撤销是已登记V1残余边界。

## 8. Spine 4.2.43 门禁

- 生产配置精确为4.2.43，禁止`latest`、`lateststable`、`4.2.xx`和beta；
- JSON声明4.2.43；主次版本与目标一致；
- 自有Spine Exporter只输出经过golden定义的JSON、透明attachment PNG与`atlas-input-manifest.json`功能子集；manifest只描述packing输入/策略，不得伪装`.atlas`；
- 无合法Professional CLI时仍可导出候选文件，但必须标`exported_unverified`；
- 有CLI时先离线probe；每个验证进程在互斥lease内显式钉扎4.2.43、无隐式network grant，并从本轮重导产物再次证明实际patch后做语义比较；
- CLI非零、超时或版本不符不会覆盖旧产物；
- `.atlas/.spine/.skel`只能由用户本地合法Professional CLI 4.2.43在本轮operation的CLI-owned目录生成，逐文件绑定command/evidence/hash；内置writer写入探针必须为0，旧文件/人工放入文件不能冒充；
- `F2S-FIX-SPINE-001`及六个golden同时断言内置JSON/透明PNG/atlas-input manifest确定性、CLI-disabled时三种专有扩展名数量为0，以及CLI-enabled变体的writer provenance；
- PSD自动化只承诺本工具写出和Editor UI导入，不声明未公开CLI能力。

## 9. 内容和动画门禁

- 每个clip具有MotionSpec、表征策略、素材覆盖、提示词、人工门和测试；
- 强透视、新视面、短暂极端轮廓不能由mesh无界硬拉；
- 关键帧图只作参考，导入后必须对齐和身份一致性检查；
- hit frame、hitbox、cancel window、root motion、socket/event均为用户批准真值；
- 不断新增corrective达到阈值时切换pose/sequence/new rig或判越界；
- 独立动作之外还测试混合、中断、取消、镜像和draw order。

## 10. UI、可访问性与渲染门禁

- 所有页面有空、忙、成功、错误、取消、只读、stale状态；
- 禁用动作显示原因，破坏性操作显示影响和可恢复方式；
- 非画布核心路径可键盘操作；焦点可见；状态不只靠颜色；
- 100/125/150/200%DPI坐标、拾取和布局正确；
- Pixi渲染循环与React状态分离，拖拽结束才提交领域Command；
- WebGL context lost、纹理预算和低性能模式可恢复/可解释；
- AI生成区、透明洞、拉伸、slot伪色、骨骼/pivot/接触点可视化。

## 11. 工程与交付门禁

- Git、锁文件、标准脚本、format/lint/typecheck/test/build可从干净checkout运行；
- npm/Cargo/uv版本精确锁定，不存在浮动Git branch或生产`latest`；
- 根目录双击入口只启动正式应用或给出诊断，不联网下载、不修改执行策略、不闪退；
- 安装、升级、降级、修复、卸载保留用户项目；
- 中文、空格、长路径、普通用户权限和干净Windows机器smoke通过；
- AI Runtime Pack与Desktop Core独立版本、SBOM和卸载边界；
- 未签名内部包明确标记，公开发布需要代码签名决策。

## 12. 明确禁止的设计内容

1. 把“任意单图自动生成商业级重度动作Spine”作为验收承诺；
2. 向第三方公网服务上传角色、prompt或项目遥测；
3. 捆绑Spine Runtime/Editor或保存用户激活材料；
4. 以内置writer逆向、生成或改写`.atlas/.spine/.skel`，或把无本轮4.2.43 CLI provenance的同名文件列为产品产物；
5. 让AI自动批准母版、关键姿势、命中帧或玩法事件；
6. 把索引数据库或缓存作为源资产唯一副本；
7. 用shell字符串拼接用户路径或向WebView开放通用shell；
8. 静默降分辨率、切模型、切远端或批准降级结果；
9. 把未运行的技术能力、跳过的外部测试或文档高分写成VERIFIED；
10. 通过局部润色掩盖跨文档契约、ID或许可冲突。
