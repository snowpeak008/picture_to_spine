# 路线图、里程碑、依赖与估算

## 1. 执行原则

路线图以可验证纵向成果解锁后续工作。用户已经授权完成全部开发，但外部商业许可证、代码签名证书、私有GPU凭据、真实角色素材，以及发布方签名的per-organization enrollment pack与三个独立审批credential不在该授权中；缺失时实现可测试适配器并标记外部验证状态，不能伪造成功。后两项缺失只禁用governed P2 waiver，不阻止“修复所有finding后发布”的核心路径。

估算为单个熟练工程角色的理想工作日区间，不是日历承诺；AI模型质量、PSD兼容和 Spine CLI 必须用 Spike 修正估算。

## 2. 总体依赖

```text
F2S-MILESTONE-M00 决策/Spike
 └─F2S-MILESTONE-M01 工程与桌面壳
    └─F2S-MILESTONE-M02 领域/IPC/存储
       └─F2S-MILESTONE-M03 导入/母版
          ├─F2S-MILESTONE-M04 分层/素材编辑
          │  └─F2S-MILESTONE-M05 Rig/mesh/IR
          └─F2S-MILESTONE-M06 动作规划/提示词/关键帧
                 └────┬──────────┘
                      F2S-MILESTONE-M07 时间轴/预览/闭环QA
                     └─F2S-MILESTONE-M08 PSD/PNG/Spine导出
                        └─F2S-MILESTONE-M09 安全/性能/恢复加固
                           └─F2S-MILESTONE-M10 安装/入口
                              └─F2S-MILESTONE-M11 全链验收/发布
```

`F2S-MILESTONE-M03` 是 `F2S-MILESTONE-M04` 与 `F2S-MILESTONE-M06` 的共同里程碑前置。`F2S-MILESTONE-M06` 的纯schema/fixture准备可在 `F2S-MILESTONE-M02` 内作为其已有任务的预研并行，但该里程碑exit不得在approved master/StyleSpec之前通过。`F2S-MILESTONE-M04`与`F2S-MILESTONE-M06`可在`F2S-MILESTONE-M03`后并行；`F2S-MILESTONE-M05`必须等待`F2S-MILESTONE-M04`，`F2S-MILESTONE-M07`必须同时等待`F2S-MILESTONE-M05`与`F2S-MILESTONE-M06`。接口fixture可以提前并行，里程碑exit和approved输入不能越过。

## 3. 里程碑

### F2S-MILESTONE-M00 — 需求冻结、许可证与技术 Spike（5–10日）

目标：把高风险假设变成有证据的决策。

- 复核已接受的 `F2S-ADR-TARGET-001`，冻结仅 Spine Editor 4.2.43 的目标注册表与输出契约；
- 建立许可分类、模型manifest和禁止项；
- 生成合法合成 fixture；
- 验证 Pixi mesh 编辑、PSD写出、Spine JSON最小导入格式；
- 在8GB GPU上验证小模型/代理分辨率能力；
- 在目标Windows 11上验证`windows-appcontainer-v1`与Python/CUDA/GPU/子进程的兼容性及恶意escape/egress探针；失败时冻结Core-only路线并禁止后续Worker Pack；
- 验证Tauri、Rust、Worker子进程、中文空格路径。

退出：P0 未决为零；失败项有路线切换；证据落盘。没有合法 Spine 时 JSON/CLI项允许 `EXTERNAL_UNVERIFIED`，但必须有静态 fixture。

### F2S-MILESTONE-M01 — 仓库、环境、桌面壳与开发入口（3–6日）

- 初始化 Git、workspace、锁文件、CI脚本和许可证检查；
- Tauri窗口、React壳、Pixi画布、Rust命令、Worker capability页；
- 根目录开发双击入口，不自动下载、不要求手工PATH；
- 诊断页显示WebView2、GPU、Worker、Spine路径与版本。

退出：干净checkout执行标准命令并启动；错误依赖有可操作提示。

### F2S-MILESTONE-M02 — 领域模型、IPC、Job引擎与项目存储（12–22日）

- StyleSpec/MotionSpec/RuntimeSpec/AssetSpec/RigIR；
- Command、RuleCatalog/RuleSetActivation、RequirementsPriorityBaseline/Change、FindingDraft/Defect origin floor、GateReport/DependencySnapshot/PrioritySlice+Observation/DecisionPolicy、append-only审批/waiver/clock流、revision双CAS、Job状态机；
- ActorRegistry empty genesis、OrganizationTrustEnrollment capability、ActorTrust/Revocation bundles、固定ActorAttestationPolicy/SignatureAlgorithmRegistry、WaiverProposal/三credential attestation与ClockRecovery跨epoch连续链；
- versioned schema、受控IPC、取消/重试/幂等；
- 可搬移项目目录、内容寻址artifact、原子保存、恢复和可重建索引。

退出：断电/损坏/Worker崩溃故障注入通过；Rule/Gate/decision/waiver/Actor/clock schema golden、R→ack→R+1→approve活性、治理/registry双head并发、P2→P1原子收紧、seq0→seq1 enrollment、连续多次ClockRecovery与离线到期恢复测试通过。

### F2S-MILESTONE-M03 — 图片接收、预检与母版审批（6–12日）

- PNG/JPEG/WebP读取限制、alpha/色彩/EXIF处理；
- 全身/裁切/单人/视角等预检接口；
- 母版、风格规范、武器占位和源哈希；
- 缩略图、对比、高亮、审批和否决历史。

退出：原图从不原位修改；不合规图片可解释拒绝；审批绑定target+DependencySnapshot+stable outcome，批准自身/无关project revision不自失效。

### F2S-MILESTONE-M04 — 分层、遮挡补全与素材编辑器（15–30日）

- 手工/半自动mask、图层树、前后顺序、原像素/生成区标识；
- 本地Worker小模型adapter与私有远端协议；
- 局部补全候选、复合差异、candidate/approved管理；
- brush、erase、split、merge、edge refine、重新组合。

退出：合成fixture覆盖遮挡/透明/不对称；任何AI输出必须人工批准；8GB降级路径可用。

### F2S-MILESTONE-M05 — 骨骼、mesh、weights 与 Rig IR（18–35日）

- 类人侧视骨架模板、pivot、父子关系、slot和draw order；
- 轮廓mesh、三角剖分、权重编辑、弯曲压力测试；

退出：合成角色完成最大关节扫描；无非法权重/三角形；所有编辑可撤销并保存。

### F2S-MILESTONE-M06 — 动作素材规划、提示词与关键帧工作流（10–20日）

- 固定动作集的MotionSpec模板和三阶段语义；

退出：10个clip都有提示词、素材策略、人工门和验收项；规则输出确定且可解释。

### F2S-MILESTONE-M07 — 动画时间轴、内部渲染与闭环 QA（18–35日）

- timeline、curve/key、attachment/draw order/event；

退出：固定动作集可制作和回放；关键姿势/命中帧必须审批；30/60FPS预算有证据。

### F2S-MILESTONE-M08 — Rig IR、PSD、PNG、Spine 4.2.43 导出（12–25日）

- 自有项目包、带标签PSD、独立PNG、Spine JSON serializer；

### F2S-MILESTONE-M09 — 安全、隐私、许可、性能与恢复加固（10–22日）

### 3.2 估算解释

顺序理想工作量合计约123–245角色日，覆盖Product/Tech Art、前端渲染、Rust核心、AI Worker、QA、安全/发布及离线组织多签治理角色。它不是日历工期；单一执行者不能把角色日直接当并行日。每个里程碑结束用真实完成时间、返工和缺陷率重估剩余范围；估算记录写入该里程碑最后一个任务EVD：F2S-EVD-M00-005、F2S-EVD-M01-005、F2S-EVD-M02-008、F2S-EVD-M03-006、F2S-EVD-M04-007、F2S-EVD-M05-008、F2S-EVD-M06-008、F2S-EVD-M07-008、F2S-EVD-M08-008、F2S-EVD-M09-007、F2S-EVD-M10-006、F2S-EVD-M11-004。

首个贯穿导出边界的纵向切片在`F2S-MILESTONE-M08`结束时交付；`F2S-MILESTONE-M00`、`F2S-MILESTONE-M01`、`F2S-MILESTONE-M02`、`F2S-MILESTONE-M03`只交付各自真实上游能力，不能提前宣称分层、Rig、预览或JSON已完成：

该切片不以漂亮样例为目标，而验证架构、数据所有权、Undo、审批、保存和导出边界。

## 5. 进度与变更控制

- 任务状态仅允许 `planned/ready/in_progress/blocked/verified/done`；
- 每次只允许一个关键路径任务处于 `in_progress`，独立模块可并行；
- 需求变化先修改 canonical 文档、影响矩阵和ADR，再改任务；
- 里程碑不能因接近时间预算而跳过测试；
- 未通过门禁的功能可隐藏在feature flag后，但不得计入完成；
- 每个里程碑结束依据实测更新剩余估算和风险。

## 6. 产品完成定义

“完成开发”要求：Windows正式构建可双击运行；固定V1动作和人工审批流程可用；项目可恢复；Rig IR/PSD/PNG/Spine 4.2.43候选JSON可导出；合法Professional环境可选验证；发布包和模型无禁止许可证；文档、测试和证据完整。外部许可证或硬件缺失只允许对应集成标记未验证，不允许伪装为已验证。
