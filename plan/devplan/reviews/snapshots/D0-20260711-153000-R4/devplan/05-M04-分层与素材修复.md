# M04 分层与素材修复原子开发计划

## 1. 里程碑合同

本文件只定义 M04 的 7 张 DEV 任务卡。它接收已人工批准且未失效的母版，产出可重放、可追溯、可人工修订并经人工批准的 LayerSet。任何本地或远程 AI 结果都只是 candidate；AI、Worker、脚本、重组分数和管理员身份均不得签发批准。完全手工分层必须在无网络、无 GPU、AI 禁用或 AI 失败时保持可用。

M04 的退出条件是：可见源像素保护策略生效，隐藏补全来源完整，重组 QA 可解释，外部重载可检测失效，且 Layer Gate 只接受当前 revision 上的有效人工批准。计划期不创建证据；执行期只允许按任务写入自己的 evidence/M04/F2S-DEV-M04-nnn/ 目录。

所有执行证据严格消费 F2S-DEV-M02-001 拥有的 `schemas/src/evidence.schema.json` 唯一 `EvidenceEnvelope`；任务专属字段只放 `payload`，禁止平行顶层合同。

## 3. 原子任务卡

### 3.1 F2S-DEV-M04-001 — Layer模型

1. **任务头**
   - DEV ID：F2S-DEV-M04-001；短名：Layer模型。
   - 目标：建立 Layer、LayerSet、像素来源、层级、可见性、边界、revision 与 candidate/approved 分离的不变量。
   - 明确非目标：不做分割推理、不做 UI、不批准 LayerSet、不修改 schemas/**。
   - 估算：2d。

2. **追踪**
   - FR/NFR：F2S-FR-LAYER-003。
   - ADR：F2S-ADR-ARCH-002、F2S-ADR-PIPE-001。
   - 风险：循环父子关系、重复稳定 ID、越界画布、把 candidate 误当 approved。
   - 上游 DEV：F2S-DEV-M03-005；下游 DEV：F2S-DEV-M04-002、F2S-DEV-M04-003、F2S-DEV-M04-005。
   - P0/P1/P2 承接：P0 数据完整性与人工 Layer Gate；无可豁免项。

3. **输入**
   - 读取 M02 生成的 Project/Artifact/Revision 类型与 M03 ApprovedMasterRef；版本由 lockfile 和生成清单确定，计划期 sha256=UNVERIFIED。
   - fixture：fixtures/projects/approved-master-v1/，执行前必须记录 manifest SHA-256；缺失时 blocked_external，不得伪造。

4. **输出**
   - crates/domain/src/layers/model/**：Rust Domain 源码，owner=F2S-DEV-M04-001，不可由生成器覆盖。
   - crates/domain/tests/layer_model/**：属性与不变量测试，owner=F2S-DEV-M04-001，可由 fixture 重跑但不可自动改写断言。

5. **设计边界**
   - Domain 拥有实体、值对象和 Specification；Application/Adapter/UI 只能调用公开领域行为。
   - 使用 Aggregate、Value Object、Specification；拒绝 Active Record、UI 状态反向写领域对象、浮点数充当稳定身份。

6. **work units**

##### F2S-WU-M04-001-01 — LayerSet 聚合
   - output：LayerId、LayerNode、LayerBounds、LayerSetRevision、LayerCandidate 聚合及不变量。
   - reads：F2S-DEV-M02-003 生成类型、F2S-DEV-M03-005 的 ApprovedMasterRef、F2S-FR-LAYER-003。
   - writes：crates/domain/src/layers/model/**。
   - steps：1) 定义稳定 ID 和整数画布坐标；2) 实现无环、唯一、边界、排序校验；3) 区分 candidate 与 approval 引用；4) 增加确定性序列化入口。
   - command：npm run typecheck。
   - tests：编译期类型边界；非法父子环、重复 ID、越界层、空 revision 被拒绝。
   - evidence：evidence/M04/F2S-DEV-M04-001/F2S-WU-M04-001-01/typecheck.log 与 evidence.json。
   - dependsOn：F2S-DEV-M03-005、F2S-DEV-M02-003。
   - parallelSafety：shared-lock:layer-domain-model。
   - rollback：删除本 WU 新增模块；不迁移项目数据；恢复前一生成类型引用。
   - estimate：1d。

##### F2S-WU-M04-001-02 — 模型属性测试
   - output：LayerSet 构造、序列化往返、revision 隔离和确定性排序测试。
   - reads：F2S-WU-M04-001-01 输出、approved-master-v1 fixture。
   - writes：crates/domain/tests/layer_model/**。
   - steps：1) 建立最小/上限 fixture；2) 生成无环与恶意图；3) 验证 round-trip 字节稳定；4) 验证 approval 不随 candidate 复制。
   - command：npm test。
   - tests：F2S-TST-CONTRACT-002、F2S-TST-E2E-002 的模型前置测试全部通过。
   - evidence：evidence/M04/F2S-DEV-M04-001/F2S-WU-M04-001-02/test.log、junit.xml、fixture-hashes.json、evidence.json。
   - dependsOn：F2S-WU-M04-001-01。
   - parallelSafety：shared-lock:layer-domain-model。
   - rollback：移除本任务测试与 fixture 引用，不改共享 fixture 内容。
   - estimate：1d。

7. **正向验收**
   - 给定已批准母版与合法层树，构造后层序、边界、revision、source artifact 均确定；序列化往返保持相同规范字节。
   - F2S-TST-CONTRACT-002、F2S-TST-E2E-002 的本任务用例通过，状态只可为 candidate，不能凭构造进入 approved。

8. **负向与故障验收**
   - 环、孤儿父节点、重复 ID、负尺寸、越画布、未知 artifact/revision 必须返回结构化错误且不产生部分聚合。
   - 腐坏 fixture、取消、OOM 或掉电均不得写批准记录；重试结果必须确定。

9. **证据**
   - EVD：F2S-EVD-M04-001；目录 evidence/M04/F2S-DEV-M04-001/。
   - evidence.json 必须通过 `schemas/src/evidence.schema.json`；使用 `taskId/startedAtUtc/endedAtUtc/inputHashes/outputHashes/logRefs/reportRefs/externalBlockers/previousEvidenceRef/notRunReason/payload`，不得使用旧别名；本任务专属层级报告置于 `payload`。
   - 计划期 capabilityState=UNVERIFIED；只有真实命令、fixture hash、JUnit 和源码 hash 齐全后可置 VERIFIED。

10. **回滚**
    - 代码：按 WU 独立撤回；schema：无写入；项目：无持久化迁移；配置/外部：无副作用。

11. **完成定义**
    - format、lint、typecheck、test 通过；领域依赖检查通过；文档追踪到 FR/ADR/TST；许可证扫描无新增问题；错误可被 UI 本地化；无伪证据。

12. **退出状态**
    - 完成代码不等于能力 VERIFIED。fixture 或跨层 E2E 未执行时标记 UNVERIFIED/blocked_external，并阻断 M04 Layer Gate 与发布。

### 3.2 F2S-DEV-M04-002 — 蒙版编辑器

1. **任务头**
   - DEV ID：F2S-DEV-M04-002；短名：蒙版编辑器。
   - 目标：提供可缩放画布上的蒙版笔刷、擦除、选区与可撤销 ApplyMaskStroke 命令。
   - 明确非目标：不直接写 LayerSet、不调用 AI、不在 pointermove 期间持久化、不批准分层。
   - 估算：3d。

2. **追踪**
   - FR/NFR：F2S-FR-LAYER-002、F2S-NFR-REL-003。
   - ADR：F2S-ADR-ARCH-002、F2S-ADR-ARCH-003。
   - 风险：高频输入淹没 IPC、缩放坐标偏移、撤销粒度错误、源像素被隐式改写。
   - 上游 DEV：F2S-DEV-M04-001；下游 DEV：F2S-DEV-M04-006。
   - P0/P1/P2 承接：P0 可逆编辑、确定坐标与人工修分层；P1 大图交互可靠性。

3. **输入**
   - LayerSet candidate、RenderSnapshot、画布 DPI/zoom/pan、MaskStrokeIntent；版本/hash 在 evidence 中登记，计划期 UNVERIFIED。
   - fixture：fixtures/layers/mask-editor-v1/**，由本任务测试 owner 创建并以 manifest 固定。

4. **输出**
   - crates/application/src/layers/mask_stroke/**：Application Command/Port。
   - apps/desktop-ui/src/features/layers/mask-editor/**：React 工具面板与 Pixi intent bridge。
   - tests/integration/layers/mask_editor/**：集成测试；均 owner=F2S-DEV-M04-002。

5. **设计边界**
   - UI 只维护 draft stroke；pointerup 形成一个命令。Application 校验 expectedRevision 并提交事务；Domain 校验蒙版边界；Pixi 不持有业务真相。
   - 使用 Command、Memento、Adapter；拒绝逐点 IPC、Canvas 对数据库直写和颜色状态作为唯一反馈。

6. **work units**

##### F2S-WU-M04-002-01 — ApplyMaskStroke 命令
   - output：坐标归一化、笔划压缩、expectedRevision CAS、undo payload。
   - reads：F2S-DEV-M04-001 领域类型、F2S-DEV-M02-004 的 UnitOfWork/CommandBus 端口。
   - writes：crates/application/src/layers/mask_stroke/**、tests/integration/layers/mask_editor/command/**。
   - steps：1) 定义 intent→command 映射；2) 校验画布、半径、点数上限；3) 单事务写 candidate revision；4) 生成逆命令；5) 测试取消与冲突。
   - command：npm run test:integration。
   - tests：F2S-TST-082、F2S-TST-CONTRACT-002；一次 stroke 只产生一次提交且可完全撤销。
   - evidence：evidence/M04/F2S-DEV-M04-002/F2S-WU-M04-002-01/integration.log、junit.xml、evidence.json。
   - dependsOn：F2S-DEV-M04-001、F2S-DEV-M02-004。
   - parallelSafety：shared-lock:layer-mask-editor。
   - rollback：撤回命令处理器；删除未发布 candidate revision；保留旧 revision。
   - estimate：1.5d。

##### F2S-WU-M04-002-02 — 画布工具与可访问交互
   - output：笔刷/擦除/框选 UI、快捷键、大小数值、状态文本与 intent bridge。
   - reads：F2S-WU-M04-002-01、RenderSnapshot/ViewportTransform。
   - writes：apps/desktop-ui/src/features/layers/mask-editor/**、tests/integration/layers/mask_editor/ui/**。
   - steps：1) 实现屏幕到画布坐标转换；2) pointermove 仅更新 draft；3) pointerup 发命令；4) 增加键盘/数值输入与非颜色状态；5) 覆盖高 DPI 和缩放。
   - command：npm test。
   - tests：F2S-TST-082、F2S-TST-E2E-002；1000 点输入仍为单命令，200% DPI 落点误差不超过 1px。
   - evidence：evidence/M04/F2S-DEV-M04-002/F2S-WU-M04-002-02/test.log、screenshots/**、interaction-trace.json、evidence.json。
   - dependsOn：F2S-WU-M04-002-01。
   - parallelSafety：shared-lock:layer-mask-editor。
   - rollback：移除功能路由和工具注册；不删除用户已有 mask revision。
   - estimate：1.5d。

7. **正向验收**
   - 缩放、平移、高 DPI 下笔划落在同一整数画布坐标；pointerup 仅提交一次；undo/redo 字节级恢复对应 mask。
   - UI 同时显示图标、文本和数值；F2S-TST-082、F2S-TST-CONTRACT-002、F2S-TST-E2E-002 通过。

8. **负向与故障验收**
   - 超大点列、NaN/越界坐标、陈旧 revision、并发重载、取消和 OOM 返回可恢复错误；不产生半笔划。
   - IPC 断开或掉电后重开项目只看到上一个已提交 revision；AI 不可被此工具隐式调用。

9. **证据**
   - EVD：F2S-EVD-M04-002；目录 evidence/M04/F2S-DEV-M04-002/；字段合同同 M04-001。
   - 必含坐标 trace、undo 前后 hash、高 DPI 截图、命令次数和 NOT_RUN 原因；执行前保持 UNVERIFIED。

10. **回滚**
    - 撤回 UI/Command；项目回滚到前一 LayerSet revision；schema 无改动；无远程副作用。

11. **完成定义**
    - format/lint/typecheck/test/integration 通过；键盘可操作；错误可理解；无逐点 IPC；追踪、许可与 evidence manifest 完整。

12. **退出状态**
    - 未取得真实 DPI、撤销和崩溃恢复证据时 capabilityState=UNVERIFIED，不能据此批准 LayerSet。

   - 明确非目标：不要求 Worker、不自动生成隐藏像素、不绕过重组 QA 与人工分层审批。

2. **追踪**

   - Application 编排导入、裁剪、命名和保存；UI 展示缺项；Domain 保证 LayerSet 不变量；文件解码通过 Port。

   - estimate：1.5d。

7. **正向验收**

   - EVD：F2S-EVD-M04-003；目录 evidence/M04/F2S-DEV-M04-003/；字段合同同 M04-001。

11. **完成定义**
    - 全部标准检查通过；离线手工 E2E 可复现；保存恢复、可访问性、错误信息、追踪和许可证检查完成。

12. **退出状态**
    - 未跑真实断网/禁 GPU/重开测试则 UNVERIFIED；无人工批准时 Layer Gate 始终关闭。
