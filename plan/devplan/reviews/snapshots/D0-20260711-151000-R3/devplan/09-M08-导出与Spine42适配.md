# M08 导出与 Spine 4.2.43 适配原子开发计划

## 0. 里程碑合同

- 本文件只定义 F2S-DOC-DEVPLAN-M08-001 的 8 个 DEV 与 17 个 WU；F2S-DEV-M07-008 的已审批 PublishSnapshot 是唯一业务输入。
- 唯一目标是 Spine Editor/CLI 4.2.43。内置 writer 只生成 Rig IR、最小 PSD、透明 attachment PNG、atlas-input-manifest.json、Spine JSON、PromptPack 和报告。
- 内置代码不得创建、改写或冒充 .atlas、.spine、.skel；三者只能来自用户本地合法 Professional CLI 4.2.43 的本轮 operation，并绑定逐文件 provenance。
- 无 Professional CLI 时，静态、schema、内部 round-trip 与 golden 仍执行；真实 CLI 项必须记录 NOT_RUN/EXTERNAL，candidate 保持 exported_unverified。
- 本文件不引入 Spine Runtime、Editor、激活信息，不实现 Unity、Godot、Cocos 或自研 Runtime 集成。
- 公共验证命令只使用根脚本 npm run format:check、npm run lint、npm run typecheck、npm test、npm run test:integration、npm run test:spine、npm run build:core。不存在的外部能力不得由模拟结果替代。
- 任务锁格式为 repo:path 或 project:<projectId>:export:<operationId>；同一锁只允许一个 writer。
- 全部执行证据严格消费 F2S-DEV-M02-001 拥有的 `schemas/src/evidence.schema.json` 唯一 `EvidenceEnvelope`；专属数据只进入 `payload`，M11 必须按该 schema 机械封账。
