# M02 领域、存储与协议基线原子计划

M02 建立后续里程碑只能消费、不能绕过的 schema、Domain、Application Port 与文件 Adapter。版本化 JSON 和不可变 artifact 是项目真相；redb 仅为可删除重建索引。所有治理判断使用规范化 bytes/hash、stable identity 和 append-only stream，不以 UI 状态、历史 checkedAt 或 transport revision 代替当前真值。

M02 同时冻结全项目唯一 `EvidenceEnvelope`：顶层必填 `schemaVersion`、`evidenceId`、`taskId`、`status`、`command`、`exitCode`、`startedAtUtc`、`endedAtUtc`、`runner`、`toolVersions`、`inputHashes`、`outputHashes`、`logRefs`、`reportRefs`、`capabilityState`、`externalBlockers`、`previousEvidenceRef`、`notRunReason` 与 `payload`。`payload` 只承载任务专属字段；禁止另造 `devId/startedAt/endedAt/inputs/outputs/tests/artifacts` 平行顶层合同。`NOT_RUN/EXTERNAL` 必须给出 `notRunReason` 和 `externalBlockers`；M03–M11 只消费 `schemas/src/evidence.schema.json`，并由 F2S-TST-CONTRACT-001 验证 Rust/TypeScript/封账读取一致。

## F2S-DEV-M02-001 — 版本化schema

#### 1. 任务头

- 非目标：不手抄多套 DTO，不接受宽松 ULID/未知字段/NaN/Infinity，不实现具体 use case。
- 预计：3d。

#### 2. 追踪
