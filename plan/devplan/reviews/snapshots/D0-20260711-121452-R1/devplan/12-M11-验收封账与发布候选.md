# M11 验收、封账与发布候选原子开发计划

## 0. 里程碑合同

- M11 只封账已实现能力，不补造实现、不降低阈值、不把 NOT_RUN/UNVERIFIED 改成 PASS/VERIFIED。
- 最小可签候选必须先通过无 waiver 正常路径：P0/P1、许可、安全、隐私、数据完整性、恢复、签名、更新、sandbox、Spine 真实性等硬门全部修复。
- Governed P2 waiver 是受控例外，不是发布必需依赖。真实 publisher enrollment、组织 offline root、三份独立 credential 缺失时，治理密码学仅可用 synthetic fixtures 做合同测试；真实 waiver 商业路径为 NOT_RUN/EXTERNAL。
- 真实 Spine Professional、私有 GPU、代码签名证书缺失时，相关能力维持外部状态；Release 候选不得扩大声明。
- release manifest 非自引用；引用 current coverage、policy、algorithm registry、ActorRegistry/enrollment/bundles、适用 freshness、clock、binding/evidence 的 artifact/domain 双 hash 与 sequence。
- 公共命令只使用 npm ci、npm run bootstrap:check、npm run format:check、npm run lint、npm run typecheck、npm test、npm run test:integration、npm run test:spine、npm run build:core、npm run build:ai-pack、npm run release:verify。
