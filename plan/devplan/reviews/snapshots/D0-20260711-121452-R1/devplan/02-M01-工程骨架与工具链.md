# M01 工程骨架与工具链原子计划

M01 只消费 M00 已接受且有 hash 的工具链基线；M00 仍为 UNVERIFIED 时，锁文件可以生成但 clean-VM 能力不得写 VERIFIED。Desktop Core 与可选 Worker Pack 从目录、构建图和发布图开始物理分离。

## F2S-DEV-M01-001 — 仓库与workspace骨架

#### 1. 任务头

- 目标：建立 Rust workspace、React/Tauri 应用边界和可执行的依赖方向检查。
- 非目标：不实现领域功能，不引入 AI Worker，不让 Domain 依赖 Tauri、React、redb、HTTP 或 Spine。
- 预计：2d。

#### 2. 追踪

- FR/NFR：F2S-NFR-MAINT-001；ADR：F2S-ADR-STACK-001、F2S-ADR-ARCH-001。
- 风险：循环依赖、UI 越过 Application 直访 Adapter、Core/Worker 混包。
- 上游：F2S-DEV-M00-001、F2S-DEV-M00-002；下游：F2S-DEV-M01-002、F2S-DEV-M01-003、F2S-DEV-M02-001。
- 携带：P0/P1 无；P2 F2S-R3A-B06-P2-001，必须有依赖方向自动测试。

#### 3. 输入

- M00 工具链/许可清单；F2S-IFC-001、F2S-IFC-002、F2S-IFC-003、F2S-IFC-004、F2S-IFC-005；目录 owner 表。
- schema/fixture：无；所有 manifest 初建后由 M01-002 锁版。
- 版本/hash 状态：M00 接受 baseline/inventory hash 必须存在；clean-VM 未完成前为 UNVERIFIED。

#### 4. 输出

- Cargo.toml；crates/domain/Cargo.toml；crates/application/Cargo.toml；crates/adapters/Cargo.toml；apps/desktop/src-tauri/Cargo.toml：workspace manifest，owner 本任务。
- apps/desktop-ui/package.json：UI 私有 manifest，owner 本任务。
- tools/architecture/dependency-rules.json 与 tools/architecture/check-boundaries.mjs：边界合同和检查器，owner 本任务。
- 再生性：manifest、规则和检查器均为受版本控制且不可再生的源，不可由构建产物反向覆盖。

#### 5. 设计边界

- Domain ← Application ← Adapter/UI，Tauri 仅 composition/DTO/error mapping。
- 使用 Hexagonal Architecture、Dependency Inversion、Composition Root；拒绝 service locator、UI 直访文件/网络、跨层相对路径导入。

- done 表示骨架可审查；未在 clean VM 构建时仍 UNVERIFIED。
- 依赖方向门禁失败阻断 M02 和 Core build。

#### 2. 追踪

- FR/NFR：F2S-NFR-MAINT-001、F2S-NFR-COMPAT-001；ADR：F2S-ADR-STACK-001。
- 风险：锁文件由错误工具生成、平台传递依赖漂移、开发机缓存掩盖缺项。
- 上游：F2S-DEV-M00-001、F2S-DEV-M01-001；下游：F2S-DOC-TRACE-001 中精确工具链消费者反向矩阵。

#### 3. 输入

- docs/decisions/F2S-TOOLCHAIN-BASELINE-001.md 及其接受 hash；M01-001 manifests。
- config：npm clean install、Cargo locked、uv frozen；fixture：空缓存 clean VM。

#### 4. 输出

- 使用 Reproducible Build、Pinned Dependency、Facade command；拒绝 postinstall 下载、浮动版本、锁文件手改。
