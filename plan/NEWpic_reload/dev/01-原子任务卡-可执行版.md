# PSD / LayerPack 输入重设计原子任务卡（可执行版）

## 0. 使用说明

本文不是路线图，而是给开发执行用的原子任务卡。每张卡都必须能单独分派、单独实现、单独验收、单独回滚。执行顺序按 `NRD-001` 至 `NRD-031`，除非任务卡明确允许并行。

每个任务卡包含：

- **开发内容**：实际要做什么。
- **设计范围**：允许改哪些层、哪些文件、哪些边界不能碰。
- **实现标准**：代码、数据、安全、错误处理、UI、测试的最低标准。
- **验收标准**：完成后如何判断任务真正完成。
- **测试要求**：必须新增或更新的测试。
- **交付物**：最终应出现的文件、模块或行为。
- **回滚方式**：失败时怎么撤回，不破坏旧流程。

## 1. 全局开发标准

### 1.1 架构标准

- Domain 只定义实体、值对象、规则和纯函数，不访问文件系统、WebView、Tauri/Win32、PNG 解码器、网络或 CAS。
- Application 只编排 use case、ports、事务、revision 校验和审批失效。
- Adapter 才允许读取文件夹、解析 PNG/PSD、写 CAS、生成受限预览。
- UI 只持有草稿和显示状态，不拥有项目真相。
- Native/Rust 是文件、CAS、revision、审批和 IPC 的权威端。

### 1.2 安全标准

- WebView 不接收用户本地绝对路径。
- WebView 不直接解码原始 LayerPack 图片；只接收 Native 生成的受限预览。
- LayerPack 所有路径必须是资源包根内相对路径。
- 禁止路径穿越、盘符路径、UNC 路径、符号链接越界。
- 不上传用户图片，不调用公网图片 API。
- 不捆绑 Photoshop、Krita、GIMP、Clip Studio、模型权重或未审计二进制。

### 1.3 数据标准

- 每次导入必须绑定 manifest hash、layer PNG hash、QA report hash、project revision。
- 修改 manifest、映射、draw order、pivot、socket、layer PNG 任何一项，都必须使旧 LayerPack candidate / Layer Gate 审批失效。
- LayerPack 导入产生的是 `PENDING` LayerSet，不能自动批准。
- `single-piece-reference` 只能做参考，不允许进入正式导出门。
- `coarse-side-humanoid-v1` 必须在 UI 和导出 manifest 中标记 `COARSE_RIG`。

### 1.4 UI 标准

- 新入口必须比手绘遮罩更显眼；手绘降级为高级修补。
- 所有阻断原因必须可读，不能只显示 Rust 内部错误。
- UI 不用“自动生成可生产动画”之类承诺性文案。
- 所有按钮 disabled 状态必须对应可解释原因。
- 紧凑面板内文字不得溢出。

### 1.5 测试标准

- schema 变更必须有正反例。
- Domain 规则必须有单元测试。
- Adapter 文件读取必须有路径越界和坏图测试。
- IPC 必须有 stale revision 测试。
- UI 必须有成功路径、阻断路径和取消路径测试。
- 集成测试必须覆盖 LayerPack -> LayerSet -> Layer Gate 的最小链路。

## 2. 原子任务卡

### NRD-001 — LayerPack 术语、边界和 feature flag

**开发内容**

- 建立新输入管线的术语和开关，确保后续代码可以以 feature flag 渐进接入。
- 把“LayerPack 主路径、手绘 repair mode、无公网、无 PS 捆绑”的边界写入项目文档。

**设计范围**

- 可改：
  - `PROJECT_MEMORY.md`
  - `docs/limits/known-limitations.md`
  - `docs/user/quick-start-zh-CN.md`
  - `plan/NEWpic_reload/**`
- 不可改：
  - 产品源码行为
  - 现有 Layer Gate 审批逻辑
  - 任何构建/发布声明

**实现标准**

- 术语必须固定：LayerPack、reference-view、layer-source、master-composite、standard rig、coarse rig、repair mode。
- feature flag 名称固定为 `F2S_LAYERPACK_IMPORT`，取值 `off|on`。
- 默认状态必须写明为 `off` 或“未接入生产入口”。

**验收标准**

- 文档明确说明 LayerPack 是新主路径，手绘遮罩是高级修补。
- 文档明确说明当前不捆绑 Photoshop 和公网服务。
- 没有把 PSD/AI 能力描述为已验证生产能力。

**测试要求**

- 不需要自动测试。
- 人工检查文档中不出现“自动一键生成完整动画”的承诺。

**交付物**

- 文档补充和术语表。

**回滚方式**

- 撤回文档变更，不影响源码和项目数据。

### NRD-002 — LayerPack JSON Schema

**开发内容**

- 新增 `manifest.f2s-layerpack.json` 的 schema，作为资源包导入合同。

**设计范围**

- 可改：
  - `schemas/src/layerpack.schema.json`
  - `tests/unit/layerpack-schema.test.mjs`
- 可读：
  - `schemas/src/common.schema.json`
  - 现有 schema 生成脚本
- 不可改：
  - Domain/Application/UI
  - 现有 project schema 的语义

**实现标准**

- schema 必须定义：
  - `schemaVersion`
  - `packId`
  - `characterSlug`
  - `rigProfile`
  - `canvas`
  - `views`
  - `sources`
  - `layers`
  - `drawOrder`
  - `sockets`
  - `qaPolicy`
  - `provenance`
- `characterSlug` 只允许 `[a-z0-9_-]+`。
- `rigProfile` 只允许：
  - `standard-side-humanoid-v1`
  - `coarse-side-humanoid-v1`
  - `single-piece-reference`
- 所有 path 字段必须是资源包内相对路径，不得包含盘符、UNC、空段、`..`。
- `networkCallsMade` 必须存在，默认语义为 0。

**验收标准**

- 合法 standard manifest 通过。
- 合法 coarse manifest 通过。
- 以下情况必须失败：
  - 缺 `side` view
  - 缺 `weapon-grip` socket
  - layer path 越界
  - `characterSlug` 含中文、空格或大写
  - `rigProfile` 未知
  - `canvas.width/height <= 0`

**测试要求**

- 新增正例：
  - `valid-standard-layerpack`
  - `valid-coarse-layerpack`
- 新增反例：
  - `invalid-path-traversal`
  - `invalid-missing-side-view`
  - `invalid-missing-weapon-socket`
  - `invalid-bad-slug`

**交付物**

- `schemas/src/layerpack.schema.json`
- schema 单测。

**回滚方式**

- 删除新增 schema 和单测，不影响生成物。

### NRD-004 — 自制 LayerPack fixtures

**开发内容**

- 生成无版权的标准/粗粒度 LayerPack 测试资源包。

**设计范围**

- 可改：
  - `fixtures/layerpacks/**`
  - `tools/fixtures/generate-layerpack-fixtures.mjs`
  - `tests/unit/layerpack-fixture.test.mjs`
- 不可使用：
  - 真实角色图
  - 第三方素材
  - 用户上传图片

**实现标准**

- fixture 图片必须由脚本生成。
- PNG 必须带 Alpha。
- standard fixture 覆盖 17 个标准 role。
- coarse fixture 覆盖 7 个粗粒度 role。
- invalid fixture 至少覆盖：
  - 路径越界
  - 缺 Alpha
  - 空层
  - 缺 socket
  - 尺寸不一致
- 每个 fixture 目录包含 `hashes.sha256` 和 `LICENSES.json`。

**验收标准**

- 所有 fixture 总大小小于 5MB。
- 重新运行生成脚本，hash 稳定。
- fixture 可以被 NRD-002 schema 测试消费。

**测试要求**

- `layerpack-fixture.test.mjs` 校验文件存在、hash 可重算、无大文件。

**交付物**

- `fixtures/layerpacks/valid-standard-pack/**`
- `fixtures/layerpacks/valid-coarse-pack/**`
- `fixtures/layerpacks/invalid-*/**`

**回滚方式**

- 删除 fixture 和生成脚本。

### NRD-006 — Domain LayerPackPolicy

**开发内容**

- 实现 LayerPack 是否满足 profile 和审批前置条件的纯规则。

**设计范围**

- 可改：
  - `crates/domain/src/layerpack/policy.rs`
  - `crates/domain/tests/layerpack_policy.rs`
- 不可改：
  - Application use case
  - Adapter 预检

**实现标准**

- `standard-side-humanoid-v1` 必须覆盖 17 role。
- `coarse-side-humanoid-v1` 必须覆盖 7 role。
- `single-piece-reference` 不能进入正式 Layer Gate。
- `weapon-grip` socket 必须存在，且 target layer role 必须是 `weapon`。
- role、drawOrder、socket 的校验必须返回结构化 issue，不只返回字符串。

**验收标准**

- 标准 profile 缺任意 required role 返回 BLOCKED。
- coarse profile 缺任意 required role 返回 BLOCKED。
- single-piece 尝试创建 LayerSet 返回 BLOCKED。
- weapon socket 指向 body/head 返回 BLOCKED。

**测试要求**

- 单元测试覆盖三种 profile。
- 负例必须验证 issue code。

**交付物**

- `LayerPackPolicy`。
- policy 测试。

**回滚方式**

- 删除 policy 模块，Domain model 保留。

### NRD-007 — 合成 QA 领域模型

**开发内容**

- 定义允许重叠后的 QA 结果模型，替代旧的“任何重叠都是错误”思路。

**设计范围**

- 可改：
  - `crates/domain/src/layerpack/qa.rs`
  - `crates/domain/tests/layerpack_qa.rs`
- 不可改：
  - 现有 `layers/recomposition.rs` 行为

**实现标准**

- QA severity 固定：
  - `BLOCKED`
  - `REVIEW`
  - `INFO`
- 指标至少包括：
  - required layer empty count
  - composite alpha coverage difference
  - composite RGB difference
  - bbox mismatch
  - unknown overlap area
  - halo/edge warning
- 重叠默认不是 BLOCKED，除非 policy 指定。

**验收标准**

- 空 required layer 必须 BLOCKED。
- 大面积未知重叠必须 REVIEW。
- 合成差异超过阈值按 `qaPolicy` 变成 BLOCKED 或 REVIEW。
- QA report 可 hash 绑定审批。

**测试要求**

- 单元测试覆盖 severity 聚合。
- 测试 QA report canonical hash 稳定。

**交付物**

- QA model。

**回滚方式**

- 删除 QA model，不影响旧 LayerSet。

**设计范围**

- 不可改：
  - 具体文件读取 adapter

**实现标准**

**测试要求**

- 使用 mock port 测试：
  - success
  - missing manifest
  - broken JSON

**交付物**

**回滚方式**

### NRD-009 — Application LayerPack 创建 LayerSet 候选

**开发内容**

- 将已通过预检的 LayerPack 转换为 `PENDING` LayerSet candidate。

**设计范围**

- 可改：
  - `crates/application/src/layerpack/create_layer_set.rs`
  - `crates/application/tests/layerpack_to_layerset.rs`
  - `crates/domain/src/layers/model.rs` 的最小扩展
- 不可改：
  - 现有审批通过逻辑
  - 现有导出逻辑

**实现标准**

- 创建前必须校验：
  - project revision
  - approved master 存在
  - preflight report 无 BLOCKED
  - LayerPack hash 未变
- 输出 LayerSet 必须是 `PENDING`。
- Layer provenance 必须记录：
  - source kind = `LayerPack`
  - manifest hash
  - layer file hashes
  - QA report hash

**验收标准**

- 创建 candidate 不会自动批准 Layer Gate。
- 修改任意输入 hash 后 create/approve 失败。
- single-piece-reference 不允许创建正式 LayerSet。

**测试要求**

- Application test 覆盖：
  - valid standard -> PENDING LayerSet
  - valid coarse -> PENDING LayerSet + coarse marker
  - blocked report -> reject
  - stale revision -> reject

**交付物**

- create use case。

**回滚方式**

- 删除 use case，保留 preflight。

### NRD-012 — PNG / Alpha adapter 和 CAS 接入

**开发内容**

- 解码 LayerPack 透明 PNG，验证 alpha、尺寸和非空层，写入 CAS。

**设计范围**

- 可改：
  - `crates/adapters/src/layerpack/png_layers.rs`
  - `crates/adapters/tests/layerpack_png.rs`
  - `apps/desktop/src-tauri/src/ipc_host.rs` 的接线部分
- 可读：
  - `crates/adapters/src/image/**`
  - `crates/adapters/src/storage/cas.rs`
- 不可改：
  - UI 直接读 PNG

**实现标准**

- 只接受 PNG layer。
- 必须存在 Alpha channel。
- 全画布 PNG 尺寸必须等于 manifest canvas。
- 局部 PNG 若支持，必须有 offset 且 bbox 在 canvas 内。
- required layer alpha 非空。
- 写 CAS 前计算 SHA-256。

**验收标准**

- 缺 Alpha -> BLOCKED。
- 空 required layer -> BLOCKED。
- 尺寸不匹配 -> BLOCKED。
- 有效 layer 写入 CAS 并返回 hash。

**测试要求**

- PNG adapter test 覆盖 RGBA、RGB、空 Alpha、尺寸不一致、超大图片。

**交付物**

- PNG adapter。

**回滚方式**

- 删除 PNG adapter；CAS 中测试数据可保留为孤立数据。

## 3. 执行完成判定

当 `NRD-001` 到 `NRD-031` 全部完成后，必须满足以下总验收：

1. 用户不需要在工具里手绘遮罩，也能通过透明 PNG LayerPack 完成 Layer Gate。
2. PSD 是增强路径，失败时能明确降级为透明 PNG 资源包。
3. 没有 PS/Krita/GIMP/Clip Studio 的用户能用 LayerPack Studio 拼装并导出资源包。
4. 新 QA 允许受控重叠，审批绑定合成预览、draw order、pivot、socket 和 hash。
5. 所有用户资产保持本地，默认无公网请求。
6. 导出包记录 LayerPack 来源和 Rig profile。
7. 旧项目、旧手绘修补、旧导出流程不被破坏。
